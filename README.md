# OpenComp

OpenComp is a Next.js competitor research workbench for mystery-shopping pricing
and checkout funnels. It runs TinyFish browser automation against competitor
sites and your own pricing page, streams live progress back to the UI, and uses
OpenAI to synthesize the findings into pricing, checkout, and Stripe
implementation recommendations.

## What It Does

- Accepts up to five competitor URLs plus your own pricing page.
- Runs TinyFish SSE browser sessions in parallel for each target.
- Shows live browser streams and an event log while research is running.
- Synthesizes completed observations with OpenAI structured output.
- Produces competitor findings, opportunities, Stripe action ideas, and a
  review-first Codex handoff with placeholder Stripe API snippets.
- Supports Supabase email/password authentication when Supabase environment
  variables are configured.

## Tech Stack

- Next.js App Router with React 19 and TypeScript
- Tailwind CSS and shadcn-style UI primitives
- Supabase SSR auth
- TinyFish automation SSE API
- OpenAI Responses API

## Project Structure

- `app/page.tsx` renders the main OpenComp workbench.
- `features/landing/components/landing-page.tsx` provides the shell and auth
  header.
- `features/research/components/research-workbench.client.tsx` contains the
  client-side research form, SSE reader, live browser previews, and report UI.
- `app/api/research/stream/route.ts` validates requests, enforces auth when
  Supabase is configured, runs TinyFish jobs, and streams research events.
- `features/research/server/tinyfish.server.ts` calls TinyFish and parses SSE
  events.
- `features/research/server/openai-synthesis.server.ts` calls OpenAI and parses
  the structured research summary.
- `features/auth/components/*` contains login, signup, logout, and header auth
  controls.
- `lib/supabase/*` contains browser, server, and proxy Supabase clients.

## Environment Variables

Create a local `.env` or `.env.local` file. Do not commit real values.

```env
NEXT_PUBLIC_SUPABASE_URL=your-project-url
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=your-publishable-or-anon-key

# Server-side only. Used by /api/research/stream.
TINYFISH_API_KEY=your-tinyfish-api-key

# Server-side only. Used to synthesize TinyFish observations.
OPENAI_API_KEY=your-openai-api-key
OPENAI_MODEL=gpt-4o-mini
```

`OPENAI_MODEL` is optional and defaults to `gpt-4o-mini`.

Supabase variables enable authentication and session refresh. When they are not
set, the research API skips the auth check, but the login and signup flows
cannot authenticate users.

## Local Development

Install dependencies:

```bash
npm install
```

Run the dev server:

```bash
npm run dev
```

Open `http://localhost:3000`.

## Scripts

- `npm run dev` starts the Next.js development server.
- `npm run build` creates a production build.
- `npm run start` starts the production server after a build.
- `npm run lint` runs ESLint.

## Security Notes

- `.env` and `.env*.local` are ignored by git.
- `.env.example` should contain placeholders only.
- TinyFish and OpenAI keys are read only on the server.
- Stripe commands generated by the report intentionally use placeholders and
  require review before touching live Stripe resources.
